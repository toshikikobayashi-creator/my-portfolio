# Sales Outreach スキル

企業の問い合わせフォーム経由で営業をかけるための文章自動生成ツール。

## 概要

このスキルは以下のワークフローを自動化する：

```
アタックリスト読込 → 企業Webリサーチ → サービス選定 → 文章生成 → 日程提案 → 確認出力 → 送信記録
```

## ワークフロー詳細

### Step 1: アタックリストの読み込み

- Excelファイル（`.xlsx`）からリストを読み込む
- 必須カラム: `企業名/ブランド名`, `カテゴリ`, `問い合わせURL`, `優先度`, `ステータス`
- `ステータス`が「未着手」の企業のみ対象
- `優先度`「高」→「中」→「低」の順に処理

### Step 2: 企業Webリサーチ（自動）

各企業について以下を調査し、文章に反映する：

1. **公式サイト/ニュースリリース** → 最新の取り組み・イベント・新商品を把握
2. **具体的な一文を抽出** → 「御社の〇〇が△△とお見受けしました」に使う
3. **企業規模の推定** → サービス選定（HUMAN AD vs アドトラック）に使用

リサーチ結果は以下のフォーマットで内部メモとして保持：

```
企業名: ○○株式会社
カテゴリ: イベント・POPUP
最新トピック: 「〇〇展を3/15-3/30に渋谷で開催」
推定規模: 大企業 / 中小企業 / スタートアップ
推奨サービス: HUMAN AD / アドトラック / 両方提案
```

### Step 3: サービス選定ロジック

→ 詳細は `references/service-info.md` を参照

| 条件 | 推奨サービス |
|------|-------------|
| 中小企業・スタートアップ・局所的プロモ | HUMAN AD |
| 大企業・全国規模・高予算・広範囲認知 | アドトラック |
| 大企業だが局所的イベントあり | 両方提案（HUMAN AD メイン） |

### Step 4: 文章生成

→ 文章ルールは `references/message-rules.md` を参照
→ カテゴリ別ペインポイントは `references/pain-points.md` を参照

**文章構成（5パート・合計300〜500文字）：**

```
① フック（1文）: 企業固有の最新情報に触れる具体的な一文
② 恐れの提示（1-2文）: カテゴリ別ペインポイントを突く
③ 解決策の提示（2-3文）: サービスの強みで恐れを解消
④ 実績/具体性（1文）: 数字・事例で信頼性を担保
⑤ 日程提案+CTA（2-3文）: 3つの候補日時 + 返信誘導
```

### Step 5: 日程候補の自動提案

- Google Calendar連携が可能な場合 → 空き時間から自動取得
- 連携なしの場合 → 現在日時から3営業日以降の候補を3つ生成
- フォーマット: `3月14日（金）14:00〜16:00` のように曜日付き
- 時間帯: 10:00〜12:00 / 13:00〜15:00 / 14:00〜16:00 の2時間枠が基本

### Step 6: 出力と確認

1社ずつ以下のフォーマットで出力し、ユーザー確認を待つ：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📧 営業文 #1: ソニー・ミュージックエンタテインメント
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🏢 カテゴリ: 新曲・音楽
🎯 推奨サービス: HUMAN AD
🔗 送信先: https://www.sonymusic.co.jp/support/

【件名】
〇〇〇〇〇〇

【本文】
〇〇〇〇〇〇〇〇〇〇

【送信者情報】
株式会社HUMAN AD
PM 小林俊輝
TEL: 070-2156-0103
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 送信する → 「OK」
✏️ 修正する → 修正指示を入力
⏭ スキップ → 「スキップ」
```

### Step 7: 送信記録（トラッキング）

送信完了後、Excelファイルを自動更新：

| カラム | 内容 |
|--------|------|
| ステータス | 「送信済み」に更新 |
| 送信日 | 送信日を記録 |
| 送信サービス | HUMAN AD / アドトラック |
| 返信有無 | 後から手動更新（未返信/返信あり/商談化） |
| 送信文面 | 送った文章を記録（改善分析用） |

### Step 8: アタックリスト自動更新

- 月1回 or ユーザー指示で新規ターゲット企業を自動調査
- Web検索で以下を探す:
  - 新規オープン店舗・イベント
  - 新商品ローンチ情報
  - POP-UP / フェス開催情報
- 既存リストと重複チェック後、新規企業を追加提案

## 重要ルール

### 絶対に守ること
1. **1社1文面** — テンプレートの使い回し厳禁。必ず企業固有の情報を入れる
2. **恐れ → 解決の流れ** — いきなりサービス紹介しない。まず相手の課題を突く
3. **具体性** — 「効果的な広告」ではなく「渋谷駅周辺で1日300人の目に触れる」
4. **日程は3つ** — 選択肢を与えることで返信のハードルを下げる
5. **300〜500文字** — 長すぎると読まれない、短すぎると伝わらない
6. **送信前に必ず確認** — 自動送信はしない

### 避けること
1. ❌ 「突然のご連絡失礼いたします」で始めない（読み飛ばされる）
2. ❌ 自社紹介から入らない（相手に関係ない話を先にしない）
3. ❌ 「ご検討いただけますと幸いです」で終わらない（具体的な次のアクションを示す）
4. ❌ 競合サービスの批判をしない
5. ❌ 同じ企業に同月内で2回以上送らない

## ファイル構成

```
sales-outreach/
├── SKILL.md              ← このファイル
├── references/
│   ├── message-rules.md  ← 文章生成の詳細ルール・トーン・禁止表現
│   ├── pain-points.md    ← カテゴリ別ペインポイント辞書
│   └── service-info.md   ← HUMAN AD / アドトラックの詳細情報・料金・強み
├── templates/
│   └── examples.md       ← カテゴリ別の文章サンプル（Few-shot用）
└── scripts/
    └── tracker.py        ← Excel読み書き・トラッキング用スクリプト
```

必要に応じて各referenceファイルを読み込んで使用すること。
